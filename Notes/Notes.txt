1. Requirement Gathering, Schema design, Entity Relationship, and API Design for AirBnb Project
    =================================================
    HOTEL BOOKING SYSTEM (HLD + LLD)
    =================================================

    Imagine:

    You open MakeMyTrip or Booking.com
    and search:

    "Hotels in Goa from 10 June to 12 June"

    System should:
        - Show available hotels
        - Show room prices
        - Allow booking
        - Take payment
        - Prevent overbooking

    This whole flow is our Hotel Booking System.


    =================================================
    HLD
    =================================================


        =================================================
        1. ACTORS
        =================================================

            1. Hotel Manager (Hotel chalane wala) :

                Responsibilities:
                    - Add hotel
                    - Add rooms
                    - Update room count
                    - Manage bookings

                Example:
                    Taj Hotel owner adds 20 Deluxe rooms.

        -------------------------------------------------

            2. Guest (Customer) :

                Responsibilities:
                    - Search hotels
                    - View rooms
                    - Book rooms
                    - Make payment

                Example:
                    Shikhil wants to book a room in Goa.


        =================================================
        2. HOTEL SEARCH SYSTEM
        =================================================

            Purpose : Find available hotels.

            Guest enters:
                - City
                - Check In Date
                - Check Out Date
                - Number Of Rooms

            Example:
                City = Goa
                CheckIn = 10 June
                CheckOut = 12 June
                Rooms = 2

            System checks Inventory table and returns available hotels.

            Flow:
                Guest
                 ↓
                Search Hotels
                 ↓
                Inventory Check
                 ↓
                Available Hotels


        =================================================
        3. HOTEL BOOKING SYSTEM
        =================================================

            Purpose : Create booking.

            Flow:
                Search Hotel
                 ↓
                Select Hotel
                 ↓
                Select Room
                 ↓
                Check Availability
                 ↓
                Add Guests
                 ↓
                Payment
                 ↓
                Booking Confirmed

            Example:
                Shikhil selects:
                    Hotel = Taj Goa
                    Room = Deluxe

            System verifies room availability.

            If available:
                    Booking is created.


        =================================================
        4. DYNAMIC PRICING
        =================================================

            Purpose : Change room price dynamically.

            Base Price : ₹3000

            -------------------------------------------------

            Case 1:
                Occupancy High
                (Rooms almost full)

            Price:
                ₹3000 → ₹4000

            -------------------------------------------------

            Case 2:
                Holiday

            Price:
                ₹3000 → ₹5000

            -------------------------------------------------

            Case 3:
                Last Minute Booking

            Price:
                ₹3000 → ₹4500

            -------------------------------------------------

            Case 4:
                Discount Offer

            Price:

                (Har pricing rule alag class me)

2. Setup schema and Entity Relationship for the AirBnb Project

    BIG PICTURE :
        This project is a hotel booking system.
         (Yeh ek hotel booking system hai.)

        A user searches hotels, selects a room, checks availability, makes payment, and creates a booking.
         (User hotel search karta hai, room select karta hai, availability check karta hai, payment karta hai aur booking create karta hai.)

        Flow : User → Hotel → Room → Inventory → Payment → Booking


    1. User
        A registered person who uses the platform.
         (Platform use karne wala registered user.)

        Role:
            - Login karta hai
            - Hotels search karta hai
            - Bookings create karta hai
            - Guests manage karta hai

    --------------------------------------------------

    2. Guest
        A person who will stay in the hotel.
         (Hotel me rehne wala banda.)

        Role:
            - Booking ke andar stay karne wale log
            - Family members ya friends ho sakte hain
            - User inhe save karke rakh sakta hai

    --------------------------------------------------

    3. Hotel
        A hotel listed on the platform.
         (Platform par listed hotel.)

        Role:
            - Hotel details store karta hai
            - Amenities aur photos store karta hai
            - Rooms provide karta hai

    --------------------------------------------------

    4. HotelContactInfo
        Stores hotel contact information.
         (Hotel ki contact details store karta hai.)

        Role:
            - Address
            - Phone Number
            - Email
            - Location

    --------------------------------------------------

    5. Room
        Represents a room category/type.
         (Room category ko represent karta hai.)

        Role:
            - Deluxe Room
            - Suite Room
            - Standard Room

        Contains:
            - Price
            - Capacity
            - Amenities
            - Total room count

    --------------------------------------------------

    6. Inventory
        Stores date-wise room availability and pricing.
         (Date wise room availability aur pricing store karta hai.)

        Role:
            - Kitne rooms available hain
            - Kitne booked hain
            - Price kya hai
            - Booking open hai ya closed

        Most important entity for availability.
         (Room availability ka source of truth.)

    --------------------------------------------------

    7. Payment
        Represents a payment transaction.
         (Payment transaction ko represent karta hai.)

        Role:
            - Amount store karta hai
            - Transaction ID store karta hai
            - Payment status track karta hai

    --------------------------------------------------

    8. Booking
        Represents a hotel reservation.
         (Hotel reservation ko represent karta hai.)

        Role:
            - User ko Hotel se connect karta hai
            - Room reserve karta hai
            - Guests store karta hai
            - Payment se connect hota hai

        Heart of the system.
         (Poore project ki sabse important entity.)

    ==================================================
    ONE LINE SUMMARY
    ==================================================

    User → Booking karne wala user
     (Booking karne wala banda)

    Guest → Hotel me stay karne wala person
     (Hotel me rehne wala banda)

    Hotel → Property being sold
     (Jo hotel customer book karta hai)

    Room → Room category
     (Deluxe, Suite, Standard)

    Inventory → Availability manager
     (Kitne rooms bache hain ye batata hai)

    Booking → Reservation record
     (Final booking record)

    Payment → Money transaction record
     (Paise ka record)

    HotelContactInfo → Hotel contact details
     (Address aur contact information)


    ==================================================
    COMPLETE BOOKING FLOW
    ==================================================

        Step 1 :
            User searches hotels.
            (User hotel search karta hai.)

        Step 2 :
            User selects a hotel.
            (User hotel choose karta hai.)

        Step 3 :
            User selects a room type.
            (User room type select karta hai.)

        Step 4 :
            System checks Inventory.
             (System inventory check karta hai.)
            Checks:
                - Room available?
                - Price?
                - Booking open?
            (Room available hai ya nahi, price kya hai aur booking open hai ya nahi.)

        Step 5 :
            User selects guests.
            (User stay karne wale guests select karta hai.)

        Step 6 :
            Booking record is created.
             (Booking record create hota hai.)
            Status : RESERVED

        Step 7 :
            User makes payment.
            (User payment karta hai.)

        Step 8 :
            Payment becomes CONFIRMED.
            (Payment successful ho jata hai.)

        Step 9 :
            Booking becomes CONFIRMED.
            (Booking confirm ho jati hai.)

        Step 10 :
            Inventory bookedCount increases.
            (Booked rooms count increase ho jata hai.)

    Flow :
        -> User (User searches Hotels)
        -> Hotel (User selects a hotel)
        -> Room (Select Room type such as Deluxe or Suite)
        -> Inventory (System checks availability, price, and booking status)
        -> Guest (User selects or adds guests)
        -> Booking (Booking record created with status = RESERVED)
        -> Payment (User completes payment)
        -> Payment (Transaction ID generated and payment status updated to CONFIRMED)
        -> Booking (Complete Booking generated with after payment succeed with transaction id)
        -> Inventory (bookedCount increased and availability updated)

    -------------------------------------------------
        PRICING STRATEGY DESIGN
    -------------------------------------------------

        Purpose :
            Provide a flexible way to calculate room prices.

        Design Pattern :
            Strategy Pattern
                +
            Decorator Pattern

        Benefit :
            New pricing rules can be added without modifying existing code.

    -------------------------------------------------
        1. PricingStrategy
    -------------------------------------------------

        Purpose :
            Defines the pricing contract.

        Method :
            calculatePrice(Inventory inventory)

        Responsibility :
            - Accept inventory
            - Return calculated room price

        Flow :
            Inventory
                ↓
            PricingStrategy
                ↓
            Price

    -------------------------------------------------
        2. BasePricingStrategy
    -------------------------------------------------

        Purpose :
            Returns the room's base price.

        Example :
            Room Base Price = ₹400
            Result = ₹400

        Responsibility :
            - Starting point of pricing chain
            - Provides default room price

    -------------------------------------------------
        3. SurgePricingStrategy
    -------------------------------------------------

        Purpose :
            Applies surge factor.

        Formula :
            Price × Surge Factor

        Example :
            ₹400 × 1.50
            Result = ₹600

        Responsibility :
            - Handle demand-based pricing
            - Increase or decrease room price

    -------------------------------------------------
        4. OccupancyPricingStrategy
    -------------------------------------------------

        Purpose :
            Increase price when occupancy is high.

        Formula :
            Occupancy = Booked Rooms / Total Rooms

        Rule :
            Occupancy > 80%
                ↓
            Price + 20%

        Example :
            Occupancy = 85%
            ₹400 → ₹480

    -------------------------------------------------
        5. UrgencyPricingStrategy
    -------------------------------------------------

        Purpose :
            Increase price for near-future bookings.

        Rule :
            Booking Date : Within Next 7 Days
            Price + 15%

        Example :
            ₹400 → ₹460

    -------------------------------------------------
        6. HolidayPricingStrategy
    -------------------------------------------------

        Purpose :
            Increase price on holidays.

        Rule :
            Holiday
                ↓
            Price + 25%

        Example :
            ₹400 → ₹500

        Future Improvement :
            Connect with Holiday API

    -------------------------------------------------
        7. PricingService
    -------------------------------------------------

        Purpose :
            Central pricing engine.

        Responsibility :
            - Create pricing chain
            - Execute all pricing strategies
            - Return final room price

        Pricing Chain :
            BasePricingStrategy
                    ↓
            SurgePricingStrategy
                    ↓
            OccupancyPricingStrategy
                    ↓
            UrgencyPricingStrategy
                    ↓
            HolidayPricingStrategy
                    ↓
                Final Price

        Example :
            Base Price = ₹400
                    ↓
            Urgency +15%
                    ↓
            Holiday +25%
                    ↓
            Final Price = ₹575

    -------------------------------------------------
        HOTEL MIN PRICE
    -------------------------------------------------

        Purpose :
            Store cheapest room price
            for each hotel and date.

        Example :
            Deluxe Room   = ₹800
            Suite Room    = ₹1200
            Standard Room = ₹600

            Stored Price : ₹600

        Benefit :
            Faster hotel search

    -------------------------------------------------
        PRICING UPDATE SERVICE
    -------------------------------------------------

        Purpose :
            Update inventory prices and hotel minimum prices.

        Responsibilities :
            - Update Inventory prices
            - Update HotelMinPrice records
            - Run scheduler

        Flow :
            Hotel
                ↓
            Inventory
                ↓
            Dynamic Pricing
                ↓
            Inventory Update
                ↓
            HotelMinPrice Update

    -------------------------------------------------
        PRICING SCHEDULER

            Every Hour
